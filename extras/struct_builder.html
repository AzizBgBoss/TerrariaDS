▸ HOW TO USE
Full walkthrough — read once and you're good.
📥 Import existing struct

Paste a full typedef struct { ... } Name; block and click parse & import. The parser auto-detects:

  • Basic types: int, u8, u16, u32, bool, float
  • Fixed arrays: int drops[8]
  • Nested arrays: int dropRange[8][2]
  • Char arrays: char name[32]
  • Pointers: u16 *ptr
  • Strips // comments automatically
🔢 Int fields

Integer fields in the entry form have three sub-inputs:

  • value — a number or a #define constant like SpriteSize_32x32. Only digits, identifiers, and operators allowed — no random strings.
  • enum type — optionally pick an enum you defined to cast the value
  • enum value — pick a specific constant from that enum

The enum dropdowns override the text value when used — useful for type/AItype fields.

🔗 Linked array length

Arrays and nested arrays can be linked to an int field. In the field table, use the linked int column dropdown to pick which int field controls how many slots are active.

Example: link dropRange[8][2] to dropCount. In the entry form, only dropCount slots will be shown and included in the output — zeros beyond that are omitted. The hardcoded max is never exceeded.

🏷️ Enums

Define your enums in section 01 before adding fields. Name the enum (e.g. EntityAIType) and list values comma-separated (e.g. ENTITY_AI_NONE,ENTITY_AI_SLIME).

Enums show up as type dropdowns on int fields, and as the array base type selector for arrays. They also appear as field types for enum-typed fields.

✏️ Fill entry + output

After defining fields, the right panel shows a form. Hit ▸ generate entry to produce the { val, val, ... } initializer. Two outputs are always available:

  • typedef struct — paste into defs.h, updates live
  • entry initializer — paste into your array in vars.h

Each block has a copy button. Field names appear as inline comments in the output.

💡 NDS / TerrariaDS tips
  • Use u8 for biomes[8] to save RAM on 4MB
  • Link drops[8], dropChance[8], dropRange[8][2] all to dropCount for clean output
  • Use enum dropdowns for type and AItype — no more typos in constants
  • The const checkbox adds const prefix in the struct def — useful for static entity tables
  • Works fully offline — open from filesystem, no server needed
